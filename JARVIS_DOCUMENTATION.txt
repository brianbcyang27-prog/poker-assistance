1. OVERVIEW
================================================================================

JARVIS is a hierarchical multi-agent AI operating system built in Python.
It features a card-game-inspired hierarchy (J-K-Q-10-9...) where:

  - JARVIS (Chief Executive) receives all user requests
  - 4 Kings manage divisions (Engineering, Personal, Research, System)
  - 18 Workers execute specialized tasks
  - Workers can use real computer control, web search, and IoT devices
  - Each agent can use a different LLM model

Key capabilities:
  - Natural language conversation (text + voice)
  - Full computer control (mouse, keyboard, browser, screen)
  - Web search and content extraction
  - ESP32/Arduino device control over WiFi
  - Obsidian-style knowledge graph memory
  - 3D neural sphere visualization
  - Multi-turn conversation with context persistence

Flow:
  1. User sends message to JARVIS
  2. JARVIS analyzes intent via LLM
  3. JARVIS delegates to appropriate King(s)
  4. King plans work, selects Workers
  5. Workers execute using LLM + Tools
  6. Workers report to King
  7. King reviews, reports to JARVIS
  8. JARVIS composes final response

5. AGENT SYSTEM
================================================================================

Classes:
  - JarvisAgent: Chief executive, single point of contact
  - BaseKing: Division managers, plan + delegate + review
  - BaseWorker: Specialized executors, use LLM + Tools

Agent States:
  idle → thinking → planning → working → reviewing → idle
                                ↘ error

Worker Execution:
  1. Receive task from King
  2. Build system prompt (specialization + tool instructions)
  3. LLM generates plan with optional [TOOL: action(params)] calls
  4. ToolExecutor executes real-world actions (up to 5 iterations)
  5. LLM processes results, generates final response
  6. Self-assess confidence (0.0-1.0)
  7. Return result to King

King Workflow:
  1. Plan work via LLM (generates subtask JSON)
  2. Assemble team of Workers
  3. Delegate subtasks sequentially
  4. Review results (confidence threshold)
  5. Return summary or "needs_revision"

JARVIS Intent Analysis:
  - LLM analyzes user request
  - Determines if direct response or delegation needed
  - Creates task plan with King assignments
  - IMPORTANT: LLM is instructed NEVER to fabricate task results

7. IOT / ESP32 INTEGRATION
================================================================================

Protocol:
  - HTTP JSON over WiFi (port 80 on ESP32)
  - ESP32 runs lightweight WebServer
  - JARVIS sends: {"cmd":"ledon","payload":{},"id":"abc123"}
  - ESP32 responds: {"status":"ok","data":{},"id":"abc123"}

Discovery:
  1. mDNS auto-discovery (_jarvis-iot._tcp.local.)
  2. Manual registration via POST /api/iot/register
  3. Config file (devices.json)

Device Manager (jarvis/iot/manager.py):
  - register_device()      Add new device
  - send_command()         Send HTTP command to device
  - read_sensor()          Read sensor value
  - broadcast()            Send to all online devices
  - start_discovery()      mDNS background discovery

API Endpoints:
  GET  /api/iot/devices              List all devices
  POST /api/iot/register             Register device
  GET  /api/iot/{device_id}          Device details
  POST /api/iot/{device_id}/command  Send command
  GET  /api/iot/{device_id}/sensor/{name}  Read sensor
  POST /api/iot/broadcast            Command all devices

8. VOICE SYSTEM
================================================================================

TTS Providers:
  1. macOS (built-in) - Default, uses 'say' command
     - Pros: Always available, no setup
     - Cons: Robotic voice

  2. Kokoro (local neural) - Best quality
     - Install: pip3 install kokoro soundfile scipy loguru
     - First load takes 10-30s (torch init)
     - Voices: af_heart, af_bella, af_nicole, af_sarah, am_adam, am_michael
     - 24kHz WAV output

  3. OpenAI (cloud) - High quality
     - Requires OPENAI_API_KEY
     - Voices: alloy, echo, fable, onyx, nova, shimmer
     - MP3 output

Provider priority: kokoro > openai > macos

Voice API:
  GET  /api/voice/models     List providers and voices
  GET  /api/voice/audio/*    Serve audio files

9. MEMORY SYSTEM
================================================================================

Knowledge Graph (jarvis/brain/memory/graph.py):
  - SQLite-backed graph database
  - Node types: concept, note, entity, decision, code, tag
  - Edge types: references, co_occurs, referenced_by
  - Full-text search across all nodes
  - Stats: node count, edge count, density

Notes (jarvis/brain/memory/note.py):
  - Markdown files with YAML frontmatter
  - Bidirectional [[wiki-links]]
  - Auto-tagging
  - Stored in memory_store/notes/

Extractor (jarvis/brain/memory/extractor.py):
  - Auto-extracts concepts from conversations
  - Creates graph nodes and edges
  - Identifies decisions, code refs, file paths

Conversation Memory:
  - All chats saved to conversations table
  - LLM context persisted per session
  - Multi-turn conversation support

12. ARDUINO LIBRARY
================================================================================

Location: jarvis/iot/arduino_library/JarvisIoT/

Installation:
  Copy JarvisIoT/ folder to Arduino/libraries/
  OR use Arduino IDE Library Manager (if published)

Basic Usage:
  #include <JarvisIoT.h>

  JarvisIoT jarvis("Device Name", "WIFI_SSID", "WIFI_PASS");

  void setup() {
      jarvis.begin();

      // Register command handlers
      jarvis.onCommand("ledon", [](JsonDocument& p, JsonDocument& r) {
          digitalWrite(LED, HIGH);
          r["status"] = "ok";
      });

      // Register sensor readers
      jarvis.addSensor("temperature", []() {
          return String(analogRead(A0));
      });

      // Register actuators
      jarvis.addActuator("led");

      // Register with JARVIS server
      jarvis.registerWith("192.168.1.100", 8000);
  }

  void loop() {
      jarvis.update();
  }

Examples included:
  - BasicCommand: LED on/off/toggle
  - SensorMonitor: DHT22 temperature + LDR light
  - ServoControl: Servo motors + RGB LED strip

API:
  onCommand(name, callback)     Register command handler
  addSensor(name, callback)     Register sensor reader
  addActuator(name)             Register controllable actuator
  setMetadata(key, value)       Set device metadata
  registerWith(host, port)      Register with JARVIS server
  notify(event, data)           Send event to JARVIS

Built-in commands:
  ping       Returns pong
  identify   Returns device info

13. TROUBLESHOOTING
================================================================================

Port 8000 in use:
  lsof -ti:8000 | xargs kill -9

Kokoro TTS slow first load:
  - First import takes 10-30s (torch initialization)
  - Subsequent uses are fast
  - If too slow, use macOS TTS instead

Playwright not working:
  python3 -m playwright install chromium

Mouse/keyboard not working:
  - Grant Accessibility permissions in System Preferences
  - Install cliclick: brew install cliclick

ESP32 not discovered:
  - Ensure same WiFi network
  - Check mDNS is enabled on router
  - Try manual registration via POST /api/iot/register

LLM not responding:
  - Check NVIDIA_API_KEY in .env
  - Verify API key at https://build.nvidia.com
  - Check network connectivity

Server won't start:
  cd /Users/brianyang/jarvis
  python3 run.py
