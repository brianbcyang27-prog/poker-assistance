1. PROJECT OVERVIEW
================================================================================

JARVIS is a multi-agent AI operating system with a poker-card hierarchy.
It controls the computer, solves tasks autonomously, controls ESP32/Arduino
over WiFi, and remembers all projects/context.

Author:     Brian Yang
Repository: https://github.com/brianbcyang27-prog/poker-assistance
Version:    3.3.1
Python:     3.12.13
LLM:        NVIDIA API (meta/llama-3.1-8b-instruct)

Tech Stack:
  Backend:   FastAPI, aiosqlite, uvicorn, httpx
  Frontend:  HTML5, CSS3, JavaScript (vanilla), Three.js
  Database:  SQLite with FTS5 full-text search
  TTS:       macOS 'say' command, Kokoro (lazy-loaded)
  CLI:       Rich (TUI mode)

2. SYSTEM ARCHITECTURE
================================================================================

JARVIS (J) — Top-level orchestrator
├── Engineering King (♠K) — 13 workers
├── Personal King (♥K) — 4 workers
├── Research King (♦K) — 3 workers
└── System King (♣K) — 3 workers

Each King:
  - Plans work using LLM
  - Assembles team of workers
  - Delegates subtasks
  - Reviews results
  - Reports to JARVIS

Each Worker:
  - Receives task from King
  - Calls LLM for reasoning
  - Executes tools (screen, mouse, keyboard, shell, browser, IoT, engineering)
  - Self-assesses confidence
  - Returns results to King

Data Flow:
  User → JARVIS → Intent Analysis → King → Worker → LLM + Tools → Results
                                     ↓
                              Event Bus → WebSocket → UI
                                     ↓
                              Memory Store → RAG → Future Context

4. BRAIN & MEMORY SYSTEM
================================================================================

4.1 LLM (jarvis/brain/llm.py)
    - NVIDIA API (OpenAI-compatible) with Ollama fallback
    - httpx-based HTTP client
    - Conversation history (20 messages max)
    - Session context persistence via SQLite
    - PII scrubbing on outbound messages
    - JSON response parsing with markdown fence stripping

4.2 RAG Memory (jarvis/brain/rag.py)
    - Retrieves from 5 sources:
      1. Conversations (FTS5)
      2. Knowledge Graph
      3. Skills
      4. Task History
      5. Memory Store
    - Keyword-based scoring (TF × source_weight)
    - Time decay factor
    - Token-budget-aware context assembly (3000 tokens max)

4.3 Knowledge Graph (jarvis/brain/memory/graph.py)
    - SQLite-backed graph database
    - Node types: concept, entity, note, conversation, decision, code
    - Edge types: related_to, contains, depends_on, created_from, references
    - Bidirectional traversal
    - [[wiki-link]] extraction
    - PageRank importance scoring

4.4 Knowledge Extractor (jarvis/brain/memory/extractor.py)
    - Extracts concepts (CamelCase, domain terms)
    - Extracts [[wiki-links]]
    - Extracts decisions
    - Auto-links co-occurring items
    - Creates graph nodes and edges

4.5 Note Manager (jarvis/brain/memory/note.py)
    - Obsidian-style markdown notes
    - Bidirectional [[links]]
    - Stored as .md files in memory_store/notes/
    - Creates graph nodes/edges for links

4.6 Memory Provider (jarvis/brain/memory_provider.py)
    - SQLite-backed with FTS5 full-text search
    - Memory types: CONVERSATION, MISSION, TASK, TERMINAL, CODE,
      DEMO, DOCUMENT, SKILL, DECISION, EXECUTION, NOTE
    - Tag-based filtering
    - Time-range queries

4.7 Model Router (jarvis/brain/model_router.py)
    - Keyword-based task classification
    - Multi-factor scoring:
      affinity (40%) + success_rate (30%) + latency (15%) + cost (15%)
    - EWMA performance tracking
    - Auto-registers NVIDIA + Ollama on import

4.8 Speculative Planner (jarvis/brain/speculative.py)
    - Pattern-based prediction
    - Pre-delegation of likely tasks
    - 5-minute TTL expiry
    - Accuracy tracking

4.9 Review Pipeline (jarvis/brain/review.py)
    - 5-check quality gate:
      1. Completeness
      2. Error signal detection
      3. Tool use verification
      4. Confidence check
      5. Issue identification
    - Score range: 0-1

4.10 DAG Planner (jarvis/brain/dag_planner.py)
     - Mission creation with task dependencies
     - Topological sort (Kahn's algorithm)
     - Cycle detection
     - Critical path calculation
     - Task state machine: PENDING → READY → RUNNING → COMPLETED/FAILED

4.11 Skill Evolution (jarvis/brain/skill_evolution.py)
     - A/B testing of strategy variants
     - Outcome tracking
     - Skill composition
     - Performance pruning

4.12 Agent Communication Interface (jarvis/brain/aci.py)
     - Priority message queue per agent
     - TTL-based message expiry
     - Broadcast support
     - Subscription system

4.13 Demo Learning (jarvis/brain/demo_learning.py)
     - Record user demonstrations
     - Abstract reusable patterns
     - Replay with variations
     - Outcome tracking

4.14 Observability (jarvis/brain/observability.py)
     - Metrics (counters, gauges)
     - Distributed tracing
     - Health checks

4.15 Developer Mode (jarvis/brain/developer.py)
     - Feature flags
     - Debug data storage
     - Environment inspection

4.16 Privacy Scrubber (jarvis/brain/privacy.py)
     - PII detection patterns:
       - Email addresses
       - Phone numbers (US)
       - SSN (may match dates)
       - Credit cards (may match long numbers)
       - IP addresses
       - API keys
       - File paths
       - Names ("my name is X")
     - Redaction with [REDACTED] markers
     - Allowlist support

4.17 Project Memory (jarvis/brain/project_memory.py)
     - Project registry with activity tracking
     - Resume commands generation
     - AppleScript-based resume script
     - Dynamic schema migration
